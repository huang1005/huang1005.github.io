vue生命周期

发表于 2017-11-12 | 分类于 vue

vue所有功能的实现都是围绕其生命周期进行的，在生命周期的不同阶段调用对应的钩子函数可以实现组件数据管理和DOM渲染两大重要功能。学习实例的生命周期，能帮助我们理解vue实例的运行机制，更好地利用钩子函数完成我们的业务代码。下图说明了实例的生命周期、你不需要立马弄明白所有东西、不过随着你的不断学习和使用、它的参考价值会越来越高。

生命周期的探究

初始化阶段

beforecreated：el 和 data 并未初始化
created:完成了 data 数据的初始化，el没有
beforeMount：完成了 el 和 data 初始化
mounted ：完成挂载

这里created和mounted这两个阶段在开发中比较重要：

created:bride_with_veil:我们一般会在这个阶段发送ajax请求。

mounted:construction_worker:这个阶段可以获取到dom的内容了，一般我们会在这个阶段操作dom

在这里我们可以看见挂载前和挂载完成两处的el值不一样、挂载前el值还是,这里就是应用的 Virtual DOM（虚拟Dom）技术，先把坑占住了。到后面mounted挂载的时候再把值渲染进去。

更新阶段

beforeUpdate: 在这里获取的dom内容是:更新前的内容
updated: 在这里获取的dom内容是:更新后的内容
更新阶段不会在页面加载后就触发，它会在你更新data数据后触发。

销毁阶段


销毁后 代码将不再受vue来控制了 你改变msg值后 页面上也不会有改变
